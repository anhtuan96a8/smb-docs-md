# 사용자 등록

## 아이템 설명

본 항목은 소프트웨어 사용자 목록을 관리하고 사용자에 대한 데이터 액세스 권한을 설정합니다

## 실행 설명서

1. 신규 생성 안내

작업 표시줄에서![](<../.gitbook/assets/0 (68).png>)항목을 선택합니다 -> ![](<../.gitbook/assets/1 (83).png>) 버튼을 클릭합니다 -> 다음 그림 IV.13.1과 같이 사용자 정보를 대화 상자에 입력합니다 -> **Save**을 클릭합니다.

![IV.13.1](<../.gitbook/assets/2 (35).png>)

정보를 완료하면 데이터가 다음과 같이 표시됩니다. 그림 IV.13.2.

![IV.13.2](<../.gitbook/assets/3 (12).png>)

편집 안내

* Step 1: 데이터 그리드에서 편집에 필요한 라인을 선택합니다. (그림 IV.13.2)
* Step 2: ![](<../.gitbook/assets/4 (18).png>) 버튼을 클릭합니다.
* Step 3: 대화상자의 정보 편집.
* Step 4: **Save**을 클릭하여 새 정보를 저장합니다.

1. 데이터 그리드의 설명 정보:

* 매니저: 관리직 사용자입니다. 위 단계는 하위 직급의 사용자를 확인하고 접근 권한을 허용할 수 있습니다. 기본 관리는 admin입니다.
* 사용자 성명: 소프트웨어 로그인에 사용되는 사용자 이름

![IV.13.3](<../.gitbook/assets/5 (6).png>)

비밀번호: 소프트웨어 로그인 암호

* 성명: 사용자의 전체 성명
* 그룹: 각 회사의 특정 기준에 따라 사용자를 그룹화하는데 사용됩니다.
* 접근 권한: 제 1 단계에 대한 데이터 접근 권한을 허용하는데 사용합니다. 예: 그림 IV.13.2, 유저명 **bich** 및 **huong**는 공장 X01에 속하는 직원 목록에만 접근이 가능합니다. 유저명 **tham**은 공장 X03, X05, TV03, TV05에 귀속된 데이터에 접근이 가능합니다.
