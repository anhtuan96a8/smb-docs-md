# 7.4	판매단가법

**판매단가법을 이용한 원가계산**

**1단계: 품목별 판매가격 집계**

이 단계에서는 각 완제품 코드별로 판매가격을 등록합니다. 이는 계수(coefficient)를 등록하는 방식과 유사합니다. 사용자의 편의를 위해 SmartBooks(SB)에서는 엑셀 파일을 통해 손쉽게 일괄 업로드할 수 있는 기능을 제공해야 합니다.

<figure><img src="../.gitbook/assets/image (58).png" alt=""><figcaption></figcaption></figure>

**2단계:** 해당 기간 동안 완제품 입고 수량 집계 (BOM과 동일)

이 단계에서는 **완제품 입고 전표**를 통해서만 SmartBooks(SB)에 데이터를 입력합니다.

<figure><img src="../.gitbook/assets/image (59).png" alt=""><figcaption></figcaption></figure>

**3단계: 재공품 합계 및 완제품으로의 전환(BOM과 동일)**

소프트웨어는 **재공품 자재 및 완성도율을 입력하는 별도 섹션을** 생성합니다.\
동시에, 재공품을 동등한 완제품으로 환산하기 위한 공식도 설정됩니다.

※ 참고: 하나의 제품은 여러 개의 \*\*완성도율(완성률)\*\*을 가질 수 있습니다.\
이는 제품이 생산 공정의 시작, 중간, 또는 마지막 단계에 있을 수 있기 때문입니다.

다양한 완성도율(완성률)을 적용하여 동등한 완제품을 계산한 후, 소프트웨어는 각 제품코드별로 환산된 동등 완제품 수량을 합산하여 표시합니다.\
위의 예시와 같이, 서로 다른 완성률로 재공품을 계산한 후에는 해당 코드의 동등한 완제품 총합이 아래와 같이 표시됩니다.

&#x20;

| Code     | Name                          | WIP qty a | Ratio | FG equivalent                |
| -------- | ----------------------------- | --------- | ----- | ---------------------------- |
| DUOI DEN | Plastic lamp holder  (NLH100) | 17,100.00 |       |                     7,338.20 |

각 재공품 코드(work-in-process code)는 한 줄씩 집계되어 최종 환산된 동등 완제품의 총합이 표시됩니다.\
이 단계의 결과는 다음 단계에서 미완성품(재공품)의 가치를 계산하는 데 사용됩니다.

**4단계: 당기 제조된 제품 수량 계산 (BOM 방식과 동일)**

기초재공품 + 당기 제조제품 à 제조 à 완제품 + 기말재공품

**기초재공품 + 당기 제조 = 완제품 + 기말재공품**

1\. 기초재공품 수량

2\. 당기 제조된 완제품

3\. 기말 재공품

-> 당기 제조 제품 = 완제품 + 기말재공품 – 기초재공품

스마트북에서 다음과 같이 제조제품 수량을 계산합니다.

**5단계: 판매 가격 비율에 따라 제품 원가 계산**

판매 가격에 따라 각 제품에 직접 재료비, 인건비 및 제조 간접비를 할당하는 단계입니다.

<figure><img src="../.gitbook/assets/image (63).png" alt=""><figcaption></figcaption></figure>

**6단계: 기말 재공품 계산**

기말재공품은 다음과 같이 계산합니다:

완성품환산량 \* 단가

완성품환산량은 3단계에서 결정되며, 단가는 5단계에서 결정됩니다.

<figure><img src="../.gitbook/assets/image (64).png" alt=""><figcaption></figcaption></figure>

**7단계: 당기 입고된 완제품 금액 결정**

당기 완제품 투입금액 = 기초재공품 + 당기 제조비용 - 기말재공품

입고 제품의 가치를 산출한 후, 아래의 공식을 통해 제품의 단가를 산정합니다:\
**제품 단가 = 총 입고 제품 가치 / 총 입고 완제품 수량 (2단계에서 산정됨)**
