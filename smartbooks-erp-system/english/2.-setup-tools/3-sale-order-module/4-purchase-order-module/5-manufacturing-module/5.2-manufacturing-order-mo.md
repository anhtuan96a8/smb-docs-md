# 5.2 Manufacturing Order (MO)

This module is used to make the Manufacturing Order. When the order is created, it links to the Inventory Request and Finished Goods Receipt. When the finished goods input equals to the MO quantity, the order completes and closes.

#### 5.2.1                 Search for a MO

Go to **Menu -> Manufacturing -> Manufacturing Order** to open the MO window.

#### 5.2.2                 MO

To create a MO, select **New** to open the input form.

<figure><img src="../../../../.gitbook/assets/image (54).png" alt=""><figcaption></figcaption></figure>

_**General info**_

\-     **MO No**.: auto created

\-     **SO No.**: Press F3 to get the dropdown list. Select the related SO(s)

\-   **Date**: MO date

\-     **BOM code**: the code of BOM created

\-    **BOM version**: auto loaded the default version. Press F3 to get the dropdown list

\-     **MO Quantity**: finished goods quantity, auto loaded from SO(s), changeable

\-     **Warehouse code**: Press F3 to get the dropdown list

\-    **Plant code**: Press F3 to get the dropdown list.

\-     **Machine Code**:

\-     **Molding Code**

\-     **SO No**:

\-     **Started date**: MO starts on

\-    **Ended date**: MO finishes on

\-     **PIC**: person in charge

\-     **Note**

\-     **Description VN/EN/KR**

_**Detail info**_

**Component**: the raw materials used in MO

\-     Auto loaded from BOM (Item code, Item name, UOM, Base Quantity)

\-     Planned Quantity: auto calculated based on MO quantity

\-     Actual Quantity: temporary auto calculated based on MO quantity. It can change later based on the Inventory Request.

\-     Variance Quantity: the gap between the planned and actual quantity

\-     **Total stock/Stock on line**: The quantity of raw materials (RM) remaining in stock for export.

\-     Note.

Operation (if any)

\-     Auto loaded from BOM (Operation code, Operation Name, Standard Duration)

\-     Actual Duration: based on the actual time

\-     Variance Duration: different between planned and actual time

Receipt/ Issue List:

\-     Auto loaded the Inventory Requests and Finished Goods Receipts used for that MO.

Small BOM:

\-     For BOMs with smaller production segments, when creating production orders for large finished products, the system will automatically generate additional production orders for the corresponding finished products. These child production orders will be displayed in the list of small MOs.
