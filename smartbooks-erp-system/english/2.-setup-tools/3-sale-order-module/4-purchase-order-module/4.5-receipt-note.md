# 4.5 Receipt Note

is used to record when products/goods are ready to be received into stock. Once the receipt is recorded, the related sales order will change its status from **Open** to **in Process** or **Completed**.

#### 4.5.1 Create Receipt Note

Go to **Menu -> Purchase Order -> Receipt note** to open the list. it is shown up as a grid table.

Select the Filter button to open the requirements or type directly the condition in the filter box of each fields.

<figure><img src="../../../.gitbook/assets/image (34).png" alt=""><figcaption></figcaption></figure>

_**Filter info**_

·     **Activated**: the status of the price list (active or inactive)

·    **Price List No.**: price list number (if remember)

·     **Vendor Code**: press F3 to get the dropdown list or type the code

·   **Item code**: it will show all of vendors that sell that item

·     **Validated date**: the date the price list is applied

·     **Expired date**: the date the price list is inactive.

<figure><img src="../../../.gitbook/assets/image (35).png" alt=""><figcaption></figcaption></figure>

To create a Receipt Note, click **New** to open the input form.&#x20;

_**General Info:**_

\-     **Reference No.**: automatically create, no changeable.

\-     **PO Date**: created date

\-    **Vendor code**: press F3 to get the dropdown list or type the code.

\-     **Currency code**: currency applied in this transaction (VND as default). If you choose other currency, you have to input the exchange rate.

\-     **PO No**: press F3 to get the dropdown list and choose PO

\-     **Warehouse**: press F3 to choose Warehouse

\-    **Currency ID**: currency applied in this transaction (VND as default). If you choose other currency, you have to input the exchange rate

\-    **Note:**

\-     **Description VN/EN/KR**

_**Detail info**_

If the PO number is selected, the system will display the corresponding items on the PO, and the user

needs to select the item codes. Afterward, adjust the quantity to match the actual receipt amount.

In case of entering items without using a PO number, click **New** to open the detailed entry window.

\-     **Item code**: press F3 to get the dropdown list or type the code

\-     **Vendor code**: press F3 to get the dropdown list or type the code

\-     **Warehouse**: Warehouse to receive the stocks. Auto loaded from the item list. To change it, press F3 to get the dropdown list or type the code.

\-     **UOM**: auto loaded

\-     **Tax code**: auto loaded from the item list, can edit the desired VAT code.

\-     **% Discount**: The discount percentage for the item. The discount amount is calculated automatically.

\-     **Amount**: Automatically calculated (excluding tax).

\-     **Tax Amount**: Automatically calculated

\-     **Total Amount**: Total amount (include tax)

#### 4.5.2                   Edit the Receipt Note

The **Receipt Note** for purchasing items can be edited only when it is in Draft status.\
When in other statuses, its data will be linked to other modules and locked.

·     To edit the note, click the edit icon or the note number.

·     Edit the necessary information. **Save**

#### 4.5.3 Delete/Print Receipt Note

·     To delete a **Receipt Note** for purchased items, click the delete icon and follow the instructions.

·     To delete multiple **Receipt Notes** at once, select the notes to delete and click the **Delete** button.

·     To print the **Receipt Note**, click the **Print** icon.

·     To export to Excel, select the **Receipt Notes** to delete and click the **Export data** button.
