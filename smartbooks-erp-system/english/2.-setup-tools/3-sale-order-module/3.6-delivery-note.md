# 3.6 Delivery Note

#### 3.6.1                   Create Delivery Note

The **Delivery note** is used to store information when goods/products are ready to be delivered to

the customer. Once the sales delivery note is created, the related sales order will change its status

from Open to Processing or Completed, depending on the delivery status.

<figure><img src="../../.gitbook/assets/image (11).png" alt=""><figcaption></figcaption></figure>

Go to **Menu -> Sales Order -> Delivery Note** to open the list

The list of **Delivery Note** is displayed in a table format. The user can change the display time to view

older orders or filter based on specific data fields

<figure><img src="../../.gitbook/assets/image (12).png" alt=""><figcaption></figcaption></figure>

To create new **Delivery Note**, click **New**

<figure><img src="../../.gitbook/assets/image (13).png" alt=""><figcaption></figcaption></figure>

_**General info**_

\-     **Reference No**.: create automatically. No changeable. If you want the view older Delivery Note, press F3.

\-    **Date**: the date that the Delivery Note is created.

\-     **Customer Code**: press F3 to get the dropdown list of customers or type the code. This field must be filled so that the system can load SO No. (Sales Order)

\-   **SO Numbers**: Press F3 to select from the list. It includes all sales orders that have not been delivered or invoiced for this customer. Choose the sales order number (one or multiple) → Confirm.

\-   **Description VN/EN/KR**: comments or remarks

\-     **Warehouse**: Select the warehouse (press F3 to view details).

\-     **Delivery Address**: The customer’s delivery address

\-    **Notes**

_**Detail Information**_

After selecting the sales order, the item details of the order will be displayed automatically below.\
Select the items that will be delivered in this shipment.

<figure><img src="../../.gitbook/assets/image (14).png" alt=""><figcaption></figcaption></figure>

After the item list for deliver is available:

·     **Check the item information** (Product code, Unit of measure, Sales order number, ordered quantity, Warehouse code) → **Save.**

·     In case the quantity to be delivered is different from the quantity on the order, modify the quantity on the screen and **Save**. After saving, the delivery note will be in Draft status.

·     If **Confirm** is clicked, the sales delivery note will change to **Hold** status. In this status, the quantity of items in the warehouse will be reserved, and data will be sent to accounting.

<figure><img src="../../.gitbook/assets/image (15).png" alt=""><figcaption></figcaption></figure>

#### 3.6.2                  Edit Deliver Note

Details of Delivery Note can be edited or deleted when it is in Draft status. When the note is moved

to Hold/Release status, the data is locked in other modules.

·     **Select the edit icon** or the sales order number to open the note.

·     **Edit the information** → **Save**.

#### 3.6.3                   **Delete/Print Delivery Note**

·     To delete **Delivery Note**, click the delete icon and follow the instructions.

·     To delete multiple notes at once, select the rows to delete → **Delete**.

·     To print **Delivery Note**, click the print icon.

·     To export to Excel, select the rows to export → **Export data**.

#### 3.6.4                   **Release Delivery Note**

·     Release is the status where **Delivery Note** data is linked with other departments (accounting). At this stage, there is no permission to edit/delete the note.
