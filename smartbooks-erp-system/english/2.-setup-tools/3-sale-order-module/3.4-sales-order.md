# 3.4 Sales order

Sales Order records the customer’s order information, such as item code, quantity, unit price, time, and delivery address

<figure><img src="../../.gitbook/assets/image (83).png" alt=""><figcaption></figcaption></figure>

#### 3.2.1                   Create Delivery Note.

Go to **Menu -> Sales Order -> Sales Order** to open the list

<figure><img src="../../.gitbook/assets/image (84).png" alt=""><figcaption></figcaption></figure>

The Sales Order screen is shown up as the grid table. You can change the period to check older notes or get the filter by customer or by SO number.

To create a Delivery Note, click **New**. The input screen is shown up to get the info.

<figure><img src="../../.gitbook/assets/image (85).png" alt=""><figcaption></figcaption></figure>

_**General info**_

\-     **SO No**.: create automatically. No changeable. If you want the view older Delivery Note, press F3.

\-     **Date**: the date that the Sales Order is created.

\-     **Customer Code**: press F3 to get the dropdown list of customers or type the code. This field must be filled so that the system can load SO No. (Sales Order number).

\-     **Currency**: The currency used for the transaction (default is VND). If another currency is chosen, the corresponding exchange rate must be entered.

\-     **Delivery Date**: The date when the goods are shipped from the warehouse.

\-     **Payment Term:** Press F3 to select the payment method from the list.

\-     **PIC**: The person responsible for the order.

\-     **Description VN/EN/KR**: comments or remarks

\-     **Sales Quotation**: The price is retrieved from the corresponding price list.

\-     **Contact name**: The contact person for the customer.

\-     **Phone**: The phone number of the contact person.

\-     **Email**: The email address of the contact person.

\-     **Delivery Address**: Press F3 to select the address from the customer’s address list.

\-     **PSI (Weekly Sales Plan)**: The sales plan for the week.

\-     Invoice No

_**Detail info**_

Select "**New**" to open the item entry form

There are two ways to add item details:

**Method 1**: Select multiple item codes from the item list.

<figure><img src="../../.gitbook/assets/image (86).png" alt=""><figcaption></figcaption></figure>

After selection, the user will be redirected to the Purchase Order screen to update the quantity and price accordingly

<figure><img src="../../.gitbook/assets/image (87).png" alt=""><figcaption></figcaption></figure>

**Method 2**: Enter each item code in the **Sales Order – Add New Detail** section

<figure><img src="../../.gitbook/assets/image (88).png" alt=""><figcaption></figcaption></figure>

\-     **Item code**: Press F3 to select from the list or enter the code manually.

\-    **Item Code of Customer**

\-     **ĐVT**: The unit of measure will automatically display based on the item code.

\-   **QTY Ordered**

\-     **Unit Price**: Automatically retrieved from the Price List; if not available, enter the price manually

\-     **Amount**: Automatically calculated

\-     **% Discount**: Discount percentage for each item; discount amount is automatically calculated.

\-     **Tax Code**:  Press F3 to select from the list. The tax amount will be calculated automatically.

\-   **Total Amount**: Automatically calculated

\-    **Delivery Date**:  The date when goods will be shipped, which may differ from the delivery date of other items in the order

\-     **Qty Delivery**: The quantity of the item to be delivered as listed in the order

\-    **Cancel Reason**

_**Import Item from excel file**_

If the user already has an Excel file with the list of items, they can choose to import the file:

·     **Select Template**: To download the sample file. Copy all the information into each column as per the instructions.

·     **Select Import Data**: To upload the file, navigate to the folder where the file is saved → select the file → Open.\
Usually, the Excel file will upload automatically. If it doesn't appear, click on the file name.

·     **Check for error messages** (if any) and adjust the data accordingly. Then, upload the file again. Import the data.

<figure><img src="../../.gitbook/assets/image (89).png" alt=""><figcaption></figcaption></figure>

#### 3.4.2                   Import Multiple Sales Orders

If the user has a list of multiple sales orders, they can use the Excel import function to save time on data entry.

·     **Select Template**: Download the sample file and fill in the required information (such as Sales Order and Sales Order Detail pages).

·     **Select Import**: Upload the file by navigating to the folder where the file is saved → select the file → Open.\
Usually, the Excel file will upload automatically. If it doesn’t appear, click on the file name.

·     **Check for input errors** in the "Error Info" column (if any) and adjust the information accordingly. Then, upload the file again.

·     **Upload Data**

#### 3.4.3                  Edit Sales Order

The user can edit/delete a sales order when its status is **Hold**. For other statuses (Processing or

Completed), the data is locked.

Click the edit icon or the sales order number to open the order. Adjust the general information

and/or detailed information, similar to when creating a new order. Save the changes.
