# 3.6	출고

#### 3.6.1     출고전표 생성 <a href="#toc201592051" id="toc201592051"></a>

출고전표는 상품/완제품이 고객에게 배송될 준비가 되었을 때 정보를 저장하는 데 사용됩니다. 전표가 생성되면 관련 주문의 상태가 배송 상태에 따라 미결 상태에서 처리 중 또는 완료됨으로 변경됩니다.

<figure><img src="../../.gitbook/assets/image (107).png" alt=""><figcaption></figcaption></figure>

**메뉴 -> 판매 -> 출고** 선택하여 수출전표 목록을 엽니다.

수출전표 목록은 그리드로 표시됩니다. 표시 시간을 변경하여 이전 전표를 보거나 각 필드별로 조건을 필터링할 수 있습니다.

<figure><img src="../../.gitbook/assets/image (108).png" alt=""><figcaption></figcaption></figure>

출고전표를 생성하는 경우 **신규** 버튼을 누르면 정보 입력 화면이 나타납니다.

<figure><img src="../../.gitbook/assets/image (109).png" alt=""><figcaption></figcaption></figure>

_**종합정보**_

\-        전표 수: 자동으로 생성되며 변경할 수 없습니다.

\-        증서일: 판매 청구서를 작성한 날짜

\-        고객 코드: F3 눌러 목록에서 선택 또는 코드 입력. 시스템에서 해당 주문(SO)을 다운로드하려면 고객 코드를 입력해야 합니다.

\-        SO 전표 수: F3 눌러 목록에서 선택합니다. 해당 고객의 모든 주문이 배송되지 않았거나 완전히 배송되지 않았습니다. SO 전표 번호 선택(1개 또는 다수) à 동의.

\-        창고 코드: 수출 창고를 선택합니다(자세한 내용은 F3 누르기).

\-        납품지: 고객의 배송 주소.

\-        비고.

\-        적요 VN/EN/KR: 다양한 언어 적요.

_**세부 정보**_

주문전표를 선택한 후 전표에 대한 자재의 정보가 하단에 자동으로 표시됩니다. 해당 출고에 배송될 자재를 선택합니다.

<figure><img src="../../.gitbook/assets/image (110).png" alt=""><figcaption></figcaption></figure>

출고 품목 목록이 준비된 후:

\-        자재 정보 확인(제품 코드, 계산 단위, SO 전표 번호, 주문 수량, 선적 수량, 수량, 창고 코드) -> **저장**

\-        납품 수량이 전표 수량과 다른 경우 인터페이스에서 수량을 수정하고 **저장합니다**. 저장되면 출고전표는 초안 상태가 됩니다.

\-        이때 Confirm을 누르면 출고전표가 보류 상태로 전환됩니다. 이 상태에서는 창고보유 자재 수가 그에 따라 차감되고 데이터가 회계로 푸시됩니다

<figure><img src="../../.gitbook/assets/image (111).png" alt=""><figcaption></figcaption></figure>

#### 3.6.2     출고전표 편집 <a href="#toc201592052" id="toc201592052"></a>

출고전표 세부 정보는 초안 상태에서 편집하거나 삭제할 수 있습니다. 전표가 보류/해제로 이동되는 경우 데이터가 다른 부서로 전송되어 잠깁니다.

\-        편집 아이콘 또는 전표 번호를 선택하여 전표를 엽니다.

\-        저장 정보 **-> 저장**

#### 3.6.3     출고전표 삭제/인쇄 <a href="#toc201592053" id="toc201592053"></a>

\-        출고전표를 삭제하려면 삭제 아이콘을 선택하고 안내를 따릅니다.

\-        한 번에 여러 전표를 삭제하려면 삭제할 **줄** 선택 -> **삭제.**

\-        **출고**전표를 인쇄하려면 인쇄 아니콘을 선택합니다.

\-        엑셀 파일을 내보내려면 인쇄할 줄 선택 à 내부내기.

#### 3.6.4     출고전표 발행 (release) <a href="#toc201592054" id="toc201592054"></a>

발행 (release)는 다른 부서(회계)와 연결된 출고전표 데이터의 상태입니다. 이 단계에서는 전표를 편집/삭제할 수 있는 권한이 없습니다
