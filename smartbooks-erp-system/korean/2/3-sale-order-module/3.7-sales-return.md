# 3.7 	반품

상품이 고객의 기준을 충족하지 못하는 경우 반품할 수 있습니다. 반품 품목에는 반품과 관련된 정보가 기록됩니다.

<figure><img src="../../.gitbook/assets/image (112).png" alt=""><figcaption></figcaption></figure>

#### 3.7.1     반품 전표 생성 <a href="#toc201592056" id="toc201592056"></a>

메뉴 -> 판매 -> **반품** 선택하여 반품 목록을 엽니다.

이전 섹션과 마찬가지로 반품 목록은 사용자가 열의 조건을 쉽게 필터링할 수 있도록 그리드 테이블로 표시됩니다.

<figure><img src="../../.gitbook/assets/image (113).png" alt=""><figcaption></figcaption></figure>

반품 전표를 만들려면 **신규를 선택합니다.**

<figure><img src="../../.gitbook/assets/image (114).png" alt=""><figcaption></figcaption></figure>

_**종합정보**_

\-        전표 수: 자동으로 생성되며 변경할 수 없습니다.

\-        증서일: 해당 전표가 생성된 날짜

\-        고객 코드: F3 눌러 고객 목록을 가져오거나 코드를 입력합니다. 시스템이 해당 출고전표를 모두 다운로드할 수 있도록 고객 코드를 입력해야 합니다.

\-        창고 코드: 반품된 상품을 받을 창고.

\-        수출 전표: 반품된 상품이 포함된 배송전표(F3 눌러 세부정보 참조).

\-        비고

\-        설명 VN/EN/KR

_**세부 정보**_

출고전표를 선택할 때 해당 전표에 포함된 품목 코드 목록은 자동으로 표시됩니다. 사용자는 반품 코드와 수량에 해당한 세부 사항을 재조정합니다.

자재 코드 삭제: 줄 맨 위에 있는 삭제 아이콘을 선택합니다. 여러 줄을 삭제하려면 왼쪽에 있는 사각형을 선택하여 삭제할 줄들을 선택 -> 삭제를 선택합니다.

반품 수량 수정: 편집 아이콘을 선택하고 요구에 따라 재조정합니다.

저장을 선택합니다.

#### 3.7.2     반품 전표 편집 <a href="#toc201592057" id="toc201592057"></a>

반품 전표는 보류 상태에 있을 때 편집/삭제할 수 있습니다. 상태가 Release로 전환되면 데이터가 다른 모듈과 연결되고 잠깁니다.

반품 전표의 세부 정보를 변경하는 경우

\-        편집 아이콘 또는 전표 번호를 선택하여 전표의 세부 정보를 엽니다.

\-        반품 전표의 신규 생성과 유사한 종합정보 및/또는 세부 정보 편집 à 저장.

#### 3.7.3     반품 전표 삭제/인쇄 <a href="#toc201592058" id="toc201592058"></a>

\-        반품 전표을 삭제하려면 삭제 아이콘을 선택하고 지침을 따릅니다.

\-        한 번에 여러 전표를 삭제하려면, 왼쪽에 있는 사각형 클릭을 통해 삭제할 줄 선택 -> **삭제를** 선택합니다.

\-        인쇄 아이콘을 선택하여 전표를 인쇄합니다.

\-        엑셀 파일을 내보내려면 각 줄의 왼쪽에 있는 사각형을 클릭하여 전표를 선택 -> **데이터 내보내기**를 선택합니다.

#### 3.7.4     반품 전표 발행 <a href="#toc201592059" id="toc201592059"></a>

발행 (release)는 다른 부서(회계)와 연결된 납품전표 데이터의 상태입니다.
