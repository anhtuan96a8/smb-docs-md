# 6.4 재고 실사(Stocktaking)

재고 실사는 실제 수량을 점검하고, 시스템상의 장부 수량과 비교할 때 사용합니다.\
일치하지 않는 경우, 차이 조정을 위한 작업이 필요합니다.

#### 6.4.1 재고 실사 생성

**메뉴 → 재고 관리 → 재고 실사**로 이동하여 실사 목록을 엽니다.

<figure><img src="../../../../../.gitbook/assets/image (30).png" alt=""><figcaption></figcaption></figure>

조회 기간(From date / To date)을 변경하여 이전의 실사 내역을 확인할 수 있습니다.\
새로운 재고 실사를 생성하려면 **\[신규]** 버튼을 선택하여 입력 화면을 엽니다.

<figure><img src="../../../../../.gitbook/assets/image (31).png" alt=""><figcaption></figcaption></figure>

**일반 정보**

* **BatNbr**: 자동 생성
* **실사일자 (Check Date)**
* **담당자 (PIC)**: 책임자
* **설명 (베트남어/영어/한국어)**

**상세 정보**\
&#xNAN;**\[신규]** 버튼을 선택하여 품목을 추가합니다.

* **품목 코드 (Item code)**: F3 키를 눌러 드롭다운 목록에서 선택하거나 직접 입력
* **창고 코드 (Warehouse code)**: F3 키를 눌러 드롭다운 목록에서 선택하거나 직접 입력
* **단위 (UOM)**: 자동으로 불러오기
* **장부 수량 (Stock Quantity)**: 자동으로 불러오기
* **실제 수량 (Actual Quantity)**: 실사 시 실제로 확인된 수량
* **차이 수량 (Variance quantity)**: 자동 계산됨
* **비고 (Note)**\
  → **저장(Save)**

#### 6.4.2 재고 실사 수정

재고 실사 상태가 \*\*보류(Hold)\*\*일 경우 정보 수정이 가능합니다.\
**처리됨(Posted)** 상태일 경우에는 데이터가 잠기며 수정할 수 없습니다.

재고 실사를 수정하려면 **수정 아이콘** 또는 **전표 번호**를 선택하여 상세 화면을 엽니다.\
일반 정보 및/또는 상세 정보를 새로 생성하는 방식과 유사하게 수정한 후 **저장(Save)** 합니다.

<figure><img src="../../../../../.gitbook/assets/image (32).png" alt=""><figcaption></figcaption></figure>

**재고 실사 삭제 / 출력 / 엑셀 내보내기**

* 재고 실사 또는 품목을 삭제하려면 **삭제 아이**콘을 선택합니다.
* 여러 전표 또는 여러 품목을 삭제하려면 각 행의 **왼쪽 체크박스**를 선택한 후 **\[삭제]** 클릭
* 전표를 출력하려면 각 행의 **출력 아이콘(프린트 아이콘)** 을 선택합니다.
* 엑셀 파일로 내보내려면 각 행의 **왼쪽 체크박스**를 선택한 후 **\[내보내기(Export)]** 클릭
