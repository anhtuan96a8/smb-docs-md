# 5.1	BOM 리스트

이 모듈은 기업의 **생산 공정을 관리**할 수 있도록 도와줍니다.

### 5.1        BOM 리스트

이 모듈에서는 생산 공정을 위한 **BOM**을 생성할 수 있습니다. BOM은 **제조 지시서**와 연결되며, 실제 원자재 사용량과 계획된 원자재 사용량을 비교 하는 데 사용됩니다.

<figure><img src="../../../../.gitbook/assets/image (9).png" alt=""><figcaption></figcaption></figure>

#### 5.1.1     BOM 검색

**메뉴 → 생산** → BOM으로 이동하여 **BOM** 창을 엽니다.

BOM 목록은 그리드로 표시되어 사용자가 원하는 항목별로 쉽게 필터링할 수 있습니다.

\-         **사용 여부(Activated)**: BOM 상태 (활성 또는 비활성)

\-         **품목 코드(Item code**): F3 키를 눌러 드롭다운 목록에서 선택하거나 코드를 입력

\-         **분류 코드(Class code)**: F3 키를 눌러 드롭다운 목록에서 선택하거나 코드를 입력

<figure><img src="../../../../.gitbook/assets/image (10).png" alt=""><figcaption></figcaption></figure>

#### 5.1.2     새로운 BOM 생성

새로운 BOM을 생성하는 경우 '신규' 버튼을 클릭합니다.

<figure><img src="../../../../.gitbook/assets/image (11).png" alt=""><figcaption></figcaption></figure>

**기본 정보**

* FG 코드 / 반제품 코드 (FG code / semi-FG Code): 완제품 코드, F3 키를 눌러 드롭다운 목록에서 선택하거나 코드를 입력
* BOM 버전 (BOM version): 완제품이 여러 BOM을 사용하는 경우 설정, 그렇지 않으면 VERSION1 사용
* 수량 (Quantity): 한 공정에서 생산되는 완제품 수량 (기본값은 1)
* 단위 (UOM)
* 완제품 창고 (FG Warehouse)
* 자재 창고 (Material Warehouse)
* 사용 여부 (Activated): 적용 여부 체크
* 기본 설정 (Default): 기본 BOM 여부 설정
* BOM 유형 (BOM Type)
* 외주처 (Subcontractor)
* 비고 (Note)
* 표준 원가 (Standard Costing): BOM 1단위를 생산하기 위한 기준 원가

**세부 정보**\
**구성품 (Component)**: BOM에 사용되는 원자재

.      \[신규] 버튼을 눌러 입력 화면을 엽니다.

&#x20;      o **품목 코드 (Item code)**: F3 키를 눌러 드롭다운 목록에서 선택하거나 코드 입력

&#x20;      o **단위 (UOM)**: 자동 불러오기

&#x20;      o **분류 코드 (Class code)**: 자동 불러오기

&#x20;      o **수량 (Quantity)**: 필요한 자재 수량

&#x20;      o **비고 (Note)**

&#x20;      o **스크랩 비율 (% scrap)**: 제조 중 발생하는 손실 비율

&#x20;      o **스크랩 수량 (Scrap Quantity)**: 제조 중 발생하는 손실 수량

&#x20;      o **BOM 버전 (BOM version):** Ver1 또는 Ver2

.      저장 (Save)

**작업(Operation)**: 해당 사항이 있는 경우 입력

<figure><img src="../../../../.gitbook/assets/image (12).png" alt=""><figcaption></figcaption></figure>

* **\[신규] 버튼을 선택하여 입력 화면을 엽니다.**

&#x20;      o **작업 코드(Operation code):** F3 키를 눌러 드롭다운 목록에서 선택하거나 코드를 입력

&#x20;      o **작업 순서(Sequence)**: 생산 공정의 순서

&#x20;      o **소요 시간(Duration)**: 필요한 작업 시간

&#x20;      o **비고(Note)**

&#x20;      o **작업명(Operation Name)**: 베트남어(VN)/영어(EN)/한국어(KR)로 입력

**자재 목록이 있는 경우, 엑셀 파일을 통해 가져올 수 있습니다.**

* **\[Template] 버튼 선택**: 올바른 형식의 파일을 다운로드합니다. 안내에 따라 열(Column)별로 데이터를 입력합니다 (Detail Component 및 Detail Operation 시트).
* **\[Import] 버튼 선택**: 파일 업로드 창을 엽니다. 파일 위치로 이동 → 파일 선택 → 열기(Open) 클릭
* 파일은 자동으로 업로드되며, 또는 파일명을 직접 선택할 수도 있습니다.

<figure><img src="../../../../.gitbook/assets/image (13).png" alt=""><figcaption></figcaption></figure>

#### 5.1.3     BOM 파일 업로드

사용자가 여러 BOM을 동시에 입록하고자 하는 경우.

\-       **샘플 선택**: 올바른 파일 형식을 받기 위해 선택합니다. 안내에 따라 Master, DetailComponent, DetailOperation 시트에 데이터를 입력합니다.

\-        **Import 선택**: Import Data 창을 열기 위해서입니다.

\-      **Choose 선택**: 파일이 저장된 위치로 이동하여 업로드할 파일을 선택하기 위해 -> 파일 선택 -> 열기

\-        **업로드** 선택

\-        오류 메시지 **확인** 후 수정합니다. (있는 경우)

\-       **데이터를 업로드합니다.**

<figure><img src="../../../../.gitbook/assets/image (14).png" alt=""><figcaption></figcaption></figure>

#### 5.1.4     BOM 수정

사용자는 BOM이 어떤 제조 지시서에도 사용되지 않는 경우에 BOM 상세 내용을 수정하거나 삭제할 수 있습니다.

BOM을 수정하기 위해, 수정 아이콘 또는 BOM 번호를 클릭하여 상세 화면을 엽니다.

기본 정보는 직접 수정할 수 있으며, 세부 정보는 각 항목 행의 편집 아이콘을 클릭하여 수정합니다. 완료 후 저장합니다

항목을 삭제하려면, 해당 행의 삭제 아이콘을 클릭하고 안내에 따라 진행합니다.

BOM 전체를 삭제하려면, 해당 행의 삭제 아이콘을 클릭하고 안내에 따라 진행합니다
