# 3.4.	수주서

수주서에는 자재 코드, 수량, 단가, 납품 시간 및 장소 등과 같은 고객의 주문 정보가 기록됩니다.

<figure><img src="../../.gitbook/assets/image (97).png" alt=""><figcaption></figcaption></figure>

#### 3.4.1     수주서 생성 <a href="#toc201592046" id="toc201592046"></a>

**메뉴 -> 판매 -> 수주서**를 선택하여 **전표** 목록을 엽니다. 목록은 사용자가 원하는 필드별로 쉽게 필터링할 수 있도록 인터페이스에 그리드 형태로 표시됩니다.

고정 표시는 해당월에 만든 전표를 반영하지만 사용자는 이 기간을 변경할 수 있습니다.

<figure><img src="../../.gitbook/assets/image (98).png" alt=""><figcaption></figcaption></figure>

판매 주문을 생성하는 경우 **신규를** 선택하여 입력 양식을 엽니다.

<figure><img src="../../.gitbook/assets/image (99).png" alt=""><figcaption></figcaption></figure>

_**종합정보**_

\-        수주 전표: 전표 번호 자동 생성, 변경 불가능.

\-        생성일: 전표를 만드는 날짜.

\-        고객 코드: F3 눌러 로드다운 목록의 자재 코드 선택.

\-        통화: 거래에 적용된 통화(기본값 VND). 다른 통화를 선택하는 경우 사용자는 해당 환율을 입력해야 합니다.

\-        납품일: 상품을 출고하여 배송하는 날짜

\-        결제 방법: F3 눌러 결제방법 목록 가져오기.

\-        담당자

\-        적요 VN,EN,KR: 다양한 언어로 설명

\-        견적서: 일정 견적서로부터 가저오는 판매 가격

\-        담당자

\-        전화

\-        메일

\-        납품지: F3 눌러 고객 목록의 주소 리스트 가져오기.

\-        PSI 주간: 주별 수주 계획

\-        세금계산서 번호: 주문서는 세금계산서 번호와 유사함.

_**세부 정보**_

신규를 선택하여 자재 입력 양식을 엽니다.

자재의 세부 정보를 추가하는 데 2개의 방법이 있습니다.

_**방법 1**_: 자재 목록에서 여러 자재 코드를 선택합니다.

<figure><img src="../../.gitbook/assets/image (100).png" alt=""><figcaption></figcaption></figure>

선택 후 사용자는 발주서 화면으로 리디렉션되어 해당 수량과 가격을 업데이트합니다

<figure><img src="../../.gitbook/assets/image (101).png" alt=""><figcaption></figcaption></figure>

_**방법 2**_: 주문 상세 내역에 **각 품목 코드 입력**

<figure><img src="../../.gitbook/assets/image (102).png" alt=""><figcaption></figcaption></figure>

\-        자재 코드: F3 눌러 목록에서 선택 또는 코드 입력

\-        단위: 품목 코드 기준 자동 표시하는 단위

\-        주문 수량

\-        단가: 가격표에서 자동 불러오기, 가격표가 없는 경우 단가 입력

\-        금액: 자동 계산

\-        할인율(%): 각 품목에 대한 할인 비율. 할인 금액은 자동 계산

\-        세율: F3 눌러 목록 선택. 세액은 자동 계산

\-        합계: 자동 계산

\-        배송일: 동일한 주문서의 다른 자재 코드의 배송일과 다를 수 있는 출고일

\-        납품 수량: 주문서에서 납품된 자재 수량 및 코드

\-        취소 사유.

_**엑셀 파일에서 품목 입력**_

사용자가 자재 목록이 포함된 엑셀 파일을 보유하고 있는 경우, import file를 선택 가능합니다.

\-        **샘플 파일 선택**: 샘플 파일을 받기 위하여 안내에 따라 각 열의 정보를 전체 복사합니다

\-        **데이터 입력 선택**: 파일을 업로드하는 경우, 파일저장 폴더로 이동 -> 파일 선택 -> 열기. 일반적으로 엑셀 파일은 자동으로 업로드되는 반면에 파일 이름을 클릭합니다.

\-        오류창(있는 경우)을 확인하고 데이터를 재조정한 후 파일을 다시 업로드합니다. **데이터 입력합니다**.

<figure><img src="../../.gitbook/assets/image (103).png" alt=""><figcaption></figcaption></figure>

#### 3.4.2     여러 판매 전표 입력 <a href="#toc201592047" id="toc201592047"></a>

사용자가 여러 판매 전표 목록을 보유하고 있는 경우, 엑셀 가져오기 기능을 사용하여 데이터 입력 시간을 절약할 수 있습니다.

\-        **템플릿 파일 선택**: 템플릿 파일을 가져오고 필요한 정보(판매 주문 및 판매 주문 세부 정보 시트 포함)를 입력합니다.

\-        **데이터 입력 선택**: 파일을 업로드하려면 파일저장 폴더로 이동 파일 -> 파일 선택 -> 열기. 일반적으로 엑셀 파일은 자동으로 업로드되는 반면에 파일 이름을 클릭합니다.

\-        오류 정보 열(있는 경우)에서 입력 오류를 확인하고 정보를 재조정하고 파일을 다시 로드합니다.

\-        **데이터 업로드**

<figure><img src="../../.gitbook/assets/image (104).png" alt=""><figcaption></figcaption></figure>

#### 3.4.3     판매 전표 편집 <a href="#toc201592048" id="toc201592048"></a>

사용자는 판매 전표가 **보류** 상태일 때 판매 전표를 편집/삭제할 수 있습니다. 다른 상태(처리 중 또는 완료됨)에서는 데이터가 잠깁니다.

조정 아이콘 또는 판매 전표 번호를 선택하여 전표를 엽니다. 신규 전표를 생성할 때와 유사하게 종합정보 또는/및 세부 정보를 조정한 후 **저장합니다**.
